在docker compose 部署beszel 面板

部署beszel hub

services:
  beszel:
    image: 'henrygd/beszel'
    container_name: 'beszel'
    restart: unless-stopped
    ports:
      - '8090:8090'
    volumes:
      - ./beszel_data:/beszel_data

注意: macOS 不支持 Docker 容器的 network_mode: host 配置。host 网络模式允许容器直接使用主机的网络接口，这种方式在 Linux 环境下很常见，因为容器和主机共享同一网络命名空间。但 macOS 和 Windows 的 Docker 架构不同，它们使用虚拟机来运行容器环境，无法直接访问主机的网络栈，因此不支持 host 网络模式

部署beszel-agent, 在docker部署只能监听容器状态

services:
  beszel-agent:
    image: "henrygd/beszel-agent"
    container_name: "beszel-agent"
    restart: unless-stopped
    ports:
      - '45876:45876'
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
    environment:
      PORT: 45876
      KEY: ""
